Связь Java и Minecraft

Майнкрафт работает на языке Java. Каждая новая версия игры требует более новую версию Java. Если у тебя стоит Java 8, а ты запускаешь 1.21 — всё сломается.

Файл pack.mcmeta (Наполнение)

Этот файл лежит в самом корне папки. Без него папка — просто мусор. Вот как он должен выглядеть внутри:

{ "pack": { "description": "§6Aerhold Project §7by KERDYS\n§bVersion 1.0", "pack_format": 48 } }

Разбор по фактам:

  • §6 и §b — это цветовые коды. 6 — золото, b — голубой. Так описание в меню будет цветным!
  • \n — это перенос строки.
  • pack_format — это самое важное. Если ты делаешь под 1.21, ставь 48.

Где писать код? (Namespace)

В папке data всегда должно быть ДВЕ папки:

  1. minecraft — здесь ты перезаписываешь стандартные вещи (крафты, теги).
  2. aerhold (твой ник/название) — здесь ты создаешь свои функции и биомы.

Когда вызываешь команду в игре, это выглядит так: /function aerhold:cool_effect

Структура Генерация мира Функции и Теги Конвертация в MOD

1. Анатомия Датапака

Датапак — это ZIP-архив или папка, которая переопределяет логику игры без изменения кода движка.

my_datapack/
├── pack.mcmeta (Манифест)
└── data/
    ├── minecraft/ (Переопределение ванилы)
    └── my_namespace/ (Твой контент)
        ├── functions/ (.mcfunction файлы)
        ├── worldgen/ (Биомы, шум, структуры)
        ├── recipes/ (Крафты)
        └── advancements/ (Ачивки)

pack.mcmeta: Критически важный файл. Для версии 1.20.x pack_format должен быть 15, для 1.21 — 48.

2. Deep Worldgen: Фильтрация и Шум

Самая сложная часть. Генерация строится на Density Functions (функциях плотности).

Density Functions

Это математические формулы, определяющие форму гор, пещер и равнин. Основные типы:

  • minecraft:noise — базовый шум (Perlin/Simplex).
  • minecraft:shifted_noise — шум со смещением (создает извилистые реки).
  • minecraft:weird_scaled_sampler — используется для хребтов гор.

Noise Settings

{
  "noise": {
    "min_y": -64,
    "height": 384,
    "size_horizontal": 1,
    "size_vertical": 2
  },
  "surface_rule": { ... }
}

Принцип "Шум -> Рельеф": Игра берет значения из noise_settings, прогоняет через функции плотности и накладывает surface_rules (блоки травы, камня, песка в зависимости от высоты и биома).

3. Энциклопедия Функций

  • Loot Tables: Контроль дропа. Можно сделать, чтобы из палки выпадал алмаз с шансом 0.01%.
  • Predicates: Условия. "Если игрок в воде И на нем шлем И сейчас ночь".
  • Damage Types: Создание своих видов урона (например, "магия пустоты").
  • Trim Materials: Добавление новых узоров для брони.

4. Трансформация: Datapack -> .JAR (Fabric/NeoForge)

Чтобы превратить датапак в полноценный мод, нужно использовать обертку. Самый быстрый способ — создать простой Fabric-мод.

  1. Скачай шаблон мода (Fabric Example Mod).
  2. Помести папку data из датапака в src/main/resources/ твоего проекта.
  3. В fabric.mod.json укажи ID твоего датапака как ID мода.
  4. Скомпилируй через Gradle: ./gradlew build.

Важно: JAR-мод позволяет загружать датапак автоматически при создании любого мира, что удобно для серверных сборок.

5. Архитектура Шума: Noise Router и Density Functions

В современных версиях Minecraft (1.19+) рельеф — это не просто рандом, а каскад функций. Вот как это работает на уровне "сотен строк JSON":

Основные узлы Noise Router:

  • barrier: Определяет, где кончается твердый блок.
  • fluid_level_flooded: Управляет заполнением пещер водой.
  • vein_toggle: Переключатель для генерации рудных жил.
  • continents: Глобальный шум для разделения суши и океана.

Пример сложной функции (Density Function):

{
  "type": "minecraft:interpolated",
  "argument": {
    "type": "minecraft:shifted_noise",
    "noise": "minecraft:clay_bands_offset",
    "xz_scale": 1.5,
    "y_scale": 0.0,
    "shift_x": "minecraft:island_noise",
    "shift_y": 0,
    "shift_z": "minecraft:island_noise"
  }
}

Фильтрация рельефа (Splines)

Чтобы "шум шел к рельефу", используются Cubic Splines. Это графики, которые говорят игре: "Если высота (terrain_estimator) равна 0.5, то сделай гору острой, а если -0.2 — преврати в каньон".

"points": [
  { "location": -1.0, "value": 0.1, "derivative": 0.0 },
  { "location": 0.0, "value": 0.5, "derivative": 1.0 },
  { "location": 1.0, "value": 1.2, "derivative": 0.0 }
]

Пример fabric.mod.json:

7. Система Raycasting (Лучи и Взаимодействие)

Чтобы создать "магический посох" или "скандинавское копье", которое летит точно в цель, тебе нужно запускать цикличную функцию, которая летит вперед, пока не встретит блок или сущность.

Алгоритм работы луча:

  1. Вызов функции от лица игрока.
  2. Телепортация невидимого маркера на 0.1 блока вперед.
  3. Проверка: if block ^ ^ ^ air — продолжаем путь.
  4. Проверка: if entity @e[distance=..1] — наносим урон.

Пример кода (ray.mcfunction):

# Рекурсивный вызов для имитации скорости света
execute if block ~ ~ ~ air run tp @s ^ ^ ^0.5
execute as @s at @s if entity @e[type=!player,distance=..1.5] run function my:damage_logic
execute as @s at @s if block ~ ~ ~ air run function my:ray

8. Хранение данных: Command Storage

В отличие от Scoreboard, Storage позволяет хранить целые JSON-массивы, списки и сложные объекты, которые не пропадают после перезахода. Это фундамент для систем прокачки (RPG-составляющая).

# Запись данных в глобальное хранилище
data modify storage aerhold:players CurrentQuest set value {id:"kill_frost_giant", progress:0, reward:"iron_ingot"}

# Чтение данных для проверки
execute if data storage aerhold:players {CurrentQuest:{id:"kill_frost_giant"}} run say Квест активен!

9. Фильтрация и размещение структур

Чтобы твои скандинавские постройки (например, драккары или алтари) не спавнились в пустыне, используется система Structure Sets и Biomes Tags.

  • data/minecraft/tags/worldgen/biome/has_structure/altar.json — здесь ты прописываешь список ID биомов (например, #minecraft:is_taiga), где разрешен спавн.
  • Placement: Используй linear_separation для того, чтобы структуры не кучковались в одном месте.
{
  "structures": "aerhold:viking_village",
  "placement": {
    "type": "minecraft:random_spread",
    "spacing": 32,
    "separation": 8,
    "salt": 12345678
  }
}

10. Продвинутые сущности: Item Display & Interaction

Забудь про невидимых стоек для брони. Современный стандарт — это Item Display для визуала и Interaction для регистрации ударов и кликов.

Визуализация (Item Display):

Позволяет рендерить любой предмет (или модель из ресурспака) с любым поворотом, размером и интерполяцией движения.

# Спавн кастомного моба (визуал)
summon item_display ~ ~ ~ {
  item:{id:"minecraft:netherite_axe",Count:1,tag:{CustomModelData:1001}},
  transformation:{left_rotation:[0f,0f,0f,1f],right_rotation:[0f,0f,0f,1f],translation:[0f,0.5f,0f],scale:[2f,2f,2f]},
  interpolation_duration:10,
  Tags:["aerhold_mob","visual"]
}

Хитбокс (Interaction):

Создает зону, по которой можно бить. Она передает ID атакующего в NBT attack или interaction.

# Хитбокс для моба
summon interaction ~ ~ ~ {
  width:1.5,
  height:3,
  Tags:["aerhold_mob","hitbox"]
}

11. Эффекты погружения (Post-Processing)

Для "скандинавской" атмосферы (туман, холод, кровь на экране) можно использовать vines и spyglass эффекты, переписанные в ресурспаке внутри твоего JAR.

Триггер эффекта через JSON:

{
  "condition": "minecraft:entity_properties",
  "entity": "this",
  "predicate": {
    "location": { "biome": "minecraft:frozen_ocean" }
  }
}

Это позволяет автоматически накладывать цветокоррекцию (Core Shaders), когда игрок заходит в ледяные биомы Aerhold.

12. Предикаты: Логические фильтры условий

Предикаты — это "умные" проверки, которые работают быстрее, чем execute в циклах.

# data/aerhold/predicates/is_holding_mjolnir.json
{
  "condition": "minecraft:entity_properties",
  "entity": "this",
  "predicate": {
    "equipment": {
      "mainhand": {
        "items": ["minecraft:netherite_hammer"],
        "nbt": "{aerhold_type:'mjolnir'}"
      }
    }
  }
}

13. Кастомные типы урона: Damage Types

В версии 1.20+ мы можем создавать свои причины смерти. Например, aerhold:frostbite (обморожение). Это позволяет игнорировать броню или вызывать особые сообщения о смерти.

Файл типа урона (JSON):

# data/aerhold/damage_type/frost.json
{
  "exhaustion": 0.1,
  "message_id": "frostbite",
  "scaling": "when_enchanted",
  "effects": "freezing"
}

Применение в коде:

# Нанести 5 единиц урона морозом
damage @s 5 aerhold:frost by @e[type=zombie,limit=1]

14. Loot Tables: Рандомные статы и атрибуты

Чтобы с боссов падали мечи с разным уроном, используй функцию set_attributes внутри лут-таблицы. Это создаст "сотни тысяч" комбинаций оружия без ручного прописывания каждого меча.

{
  "type": "minecraft:item",
  "name": "minecraft:netherite_sword",
  "functions": [
    {
      "function": "minecraft:set_attributes",
      "modifiers": [
        {
          "attribute": "minecraft:generic.attack_damage",
          "name": "Random Might",
          "amount": { "min": 5.0, "max": 12.0 },
          "operation": "addition",
          "slot": "mainhand"
        }
      ]
    },
    {
      "function": "minecraft:set_name",
      "name": { "text": "Меч Павшего Воина", "color": "gold", "italic": true }
    }
  ]
}

15. Оптимизация: Tick & Load Tags

Чтобы сервер не лагал от обилия кода, критически важно разделять функции на те, что работают каждый тик, и те, что работают по событию.

  • data/minecraft/tags/functions/tick.json — Только самые важные проверки (Raycasting, Таймеры).
  • data/minecraft/tags/functions/load.json — Инициализация Storage и Scoreboard при старте/перезагрузке.

16. Автоматизация сборки JAR-архива

Для профессиональной разработки используй Gradle. Вот пример блока сборки, который упакует твой датапак в мод:

// build.gradle (фрагмент)
jar {
    from("src/main/resources") {
        include "data/**"
        include "assets/**"
        include "fabric.mod.json"
        include "pack.mcmeta"
    }
}

17. Плавная анимация: Transformation Interpolation

Раньше для движения моделей нужны были сотни команд tp, что вызывало лаги. Теперь мы используем интерполяцию. Игра сама рассчитывает плавный переход между точками.

Код для плавного поворота:

# Начать поворот за 20 тиков (1 секунда)
data merge entity @e[tag=viking_statue,limit=1] {
  start_interpolation: 0,
  interpolation_duration: 20,
  transformation: {
    left_rotation: [0f, 0.707f, 0f, 0.707f],
    right_rotation: [0f, 0f, 0f, 1f],
    translation: [0f, 0f, 0f],
    scale: [1f, 1f, 1f]
  }
}

Зачем это нужно?

  • Открывающиеся ворота в крепость.
  • Вращающиеся руны вокруг алтаря.
  • Плавный взлет драккара (если это магия).

18. Звуковой ландшафт: sounds.json

Для скандинавской атмосферы нужны вопли воронов, гул ветра и боевые горны. Ты создаешь файл assets/aerhold/sounds.json внутри своего JAR.

{
  "ambient.arctic_wind": {
    "category": "ambient",
    "sounds": [
      { "name": "aerhold:ambient/wind_loop_1", "stream": true },
      { "name": "aerhold:ambient/wind_loop_2", "stream": true }
    ]
  },
  "item.horn.viking_call": {
    "category": "player",
    "sounds": [ "aerhold:item/horn_blast" ]
  }
}

Вызов в игре: playsound aerhold:ambient.arctic_wind ambient @a ~ ~ ~ 1 1

19. Кастомные чары (1.21+ Enchantment Providers)

Теперь чары — это не захардкоженные ID, а JSON-файлы. Ты можешь создать эффект "Заморозка", который срабатывает только в дождь.

# data/aerhold/enchantment/frost_aspect.json
{
  "description": {"translate": "enchantment.aerhold.frost_aspect"},
  "supported_items": "#minecraft:enchantable/sharpness",
  "weight": 5,
  "max_level": 3,
  "effects": {
    "minecraft:post_attack": [
      {
        "enchanted": "attacker",
        "affected": "victim",
        "effect": {
          "type": "minecraft:apply_mob_effect",
          "to_apply": "minecraft:slowness",
          "min_duration": 20, "max_duration": 60,
          "min_amplifier": 1, "max_amplifier": 2
        }
      }
    ]
  }
}

20. Как генерировать сотни тысяч строк?

Когда проект разрастается до масштабов Aerhold, ручной ввод JSON убивает продуктивность. Профессионалы используют Data Generators.

  • Python/Node.js: Скрипты, которые по шаблону выплевывают тысячи файлов рецептов и достижений.
  • MCTools / Datapack Helper Plus: Плагины для VS Code, которые автозаполняют NBT-теги и пути к файлам.
  • Macros в функциях: Использование $ для динамической подстановки имен игроков или координат.

21. Измерения: Создание Нифльхейма

Чтобы создать свой мир, тебе нужно два файла: dimension_type (физика: светло/темно, есть ли кровать) и dimension (генерация: биомы, шум).

Настройки типа мира:

# data/aerhold/dimension_type/niflheim.json
{
  "ultrawarm": false,
  "natural": true,
  "coordinate_scale": 1.0,
  "has_skylight": true,
  "has_ceiling": false,
  "ambient_light": 0.1,
  "monster_spawn_light_level": 0,
  "monster_spawn_block_light_limit": 0,
  "infiniburn": "#minecraft:infiniburn_overworld",
  "effects": "minecraft:overworld",
  "min_y": -64,
  "height": 384,
  "logical_height": 384
}

Генератор (Chunk Generator):

Здесь ты указываешь свой noise_router (из главы 5) и биомы. Ты можешь сделать мир, состоящий только из ледяных пустошей и каньонов.

22. Динамическая погода: Снежные бури

В Aerhold погода — это не просто декор, а угроза. Мы используем Predicates и Raycasting, чтобы проверять, находится ли игрок под открытым небом во время шторма.

# function aerhold:weather_tick
execute as @a at @s if predicate aerhold:is_exposed_to_storm run function aerhold:freeze_player

# predicate: is_exposed_to_storm
{
  "condition": "minecraft:all_of",
  "terms": [
    { "condition": "minecraft:weather_check", "is_thundering": true },
    { "condition": "minecraft:location_check", "predicate": { "can_see_sky": true } }
  ]
}

Если игрок видит небо во время грозы (в заснеженном биоме это будет метель), запускается freeze_player, которая постепенно заполняет шкалу Freezing в NBT игрока.

23. Система Баз Данных: Валюта и Репутация

Для "сотен тысяч строк" статистики игроков мы используем связку Scoreboard (для быстрых вычислений) и Storage (для долгого хранения).

  • aerhold.gold — основная валюта.
  • aerhold.reputation — отношение фракций (Викинги, Валькирии).
# Пример сложной логики покупки:
execute as @p if score @s aerhold.gold >= 500 run {
  scoreboard players remove @s aerhold.gold 500
  give @s netherite_sword{aerhold_custom:1b}
  title @s actionbar {"text":"Вы купили Меч Одина!", "color":"gold"}
}

24. Финальная упаковка: Manifest.mf

При переформатировании в .JAR убедись, что внутри архива по пути META-INF/MANIFEST.MF прописаны правильные метаданные. Это позволит лаунчерам (Fabric/Forge) распознать твой датапак как мод.

Manifest-Version: 1.0
Fabric-Loom-Version: 1.2-SNAPSHOT

25. Атмосфера: Генерация частиц (Aurora Borealis)

Для создания эффекта северного сияния мы используем облака частиц dust с переменным цветом и параметром force, чтобы их было видно издалека.

Команда для сияния:

# Генерация бирюзовых частиц на большой высоте
execute at @a run particle minecraft:dust 0.2 1.0 0.8 2 ^ ^30 ^ 10 2 10 0 100 force

Динамика цвета:

Через функции можно менять RGB-параметры пыли каждый тик, создавая эффект переливания от зеленого к фиолетовому.

26. Приват зон: Interaction & Adventure Mode

Чтобы игроки не разнесли твой спавн или квестовые локации, используй систему переключения режимов на основе Distance и Tags.

# Если игрок в радиусе 50 блоков от центра спавна и не имеет тега "admin"
execute as @a[distance=..50,tag=!admin] run gamemode adventure @s

# Если игрок вышел за пределы — возвращаем выживание
execute as @a[distance=51..,gamemode=adventure,tag=!admin] run gamemode survival @s

Это самый простой и надежный способ "заприватить" территорию в чистом датапаке.

27. Анимация появления структур

Хочешь, чтобы храм Одина "вырастал" из земли? Используй послойную загрузку через structure blocks и задержку в тиках.

  1. Сохрани постройку по слоям (layer_0, layer_1...).
  2. Запускай функцию, которая ставит block_marker.
  3. Маркер по таймеру спавнит каждый слой: place structure aerhold:temple_layer_1 ~ ~ ~.

25. Атмосфера: Генерация частиц (Aurora Borealis)

Для создания эффекта северного сияния мы используем облака частиц dust с переменным цветом и параметром force, чтобы их было видно издалека.

Команда для сияния:

# Генерация бирюзовых частиц на большой высоте
execute at @a run particle minecraft:dust 0.2 1.0 0.8 2 ^ ^30 ^ 10 2 10 0 100 force

Динамика цвета:

Через функции можно менять RGB-параметры пыли каждый тик, создавая эффект переливания от зеленого к фиолетовому.

26. Приват зон: Interaction & Adventure Mode

Чтобы игроки не разнесли твой спавн или квестовые локации, используй систему переключения режимов на основе Distance и Tags.

# Если игрок в радиусе 50 блоков от центра спавна и не имеет тега "admin"
execute as @a[distance=..50,tag=!admin] run gamemode adventure @s

# Если игрок вышел за пределы — возвращаем выживание
execute as @a[distance=51..,gamemode=adventure,tag=!admin] run gamemode survival @s

Это самый простой и надежный способ "заприватить" территорию в чистом датапаке.

27. Анимация появления структур

Хочешь, чтобы храм Одина "вырастал" из земли? Используй послойную загрузку через structure blocks и задержку в тиках.

  1. Сохрани постройку по слоям (layer_0, layer_1...).
  2. Запускай функцию, которая ставит block_marker.
  3. Маркер по таймеру спавнит каждый слой: place structure aerhold:temple_layer_1 ~ ~ ~.

28. Продвинутый ИИ: Мозги для Мобов

Чтобы мобы не просто бегали за игроком, а патрулировали территорию или "охраняли" объект, мы используем систему Brain Tags и execute facing.

Патрулирование по точкам:

Создай невидимые маркеры (Area Effect Cloud) с тегом waypoint. Моб будет идти к ближайшему маркеру, пока не найдет игрока.

# Заставить моба смотреть на точку маршрута
execute as @e[tag=guard] at @s facing entity @e[tag=waypoint,limit=1,sort=nearest] feet run tp @s ^ ^ ^0.2

Агрессия по условию:

Используй on target (1.20.2+), чтобы моб атаковал только тех, кто ударил его союзника или вошел в священную зону.

29. Эпические Битвы: Полоски здоровья и Фазы

Для боссов (например, Ледяного Великана) недостаточно просто много HP. Нужны фазы боя: на 50% здоровья он должен сменить атаку.

# Создание полоски
bossbar add aerhold:giant "Ледяной Великан"
bossbar set aerhold:giant color blue
bossbar set aerhold:giant players @a

# Логика фаз (в tick.mcfunction)
execute as @e[tag=giant_boss] store result bossbar aerhold:giant value run data get entity @s Health
execute as @e[tag=giant_boss,nbt={Health:50f}] run function aerhold:boss_phase_2

Фаза 2: Можно вызвать particle взрыв и выдать боссу эффект силы или новые способности (например, призыв волков).

30. Оптимизация JAR для релиза

Когда в твоем датапаке (уже моде) становятся тысячи файлов, игра может подтормаживать при загрузке. Как это лечить?

  • Сжатие JSON: Используй минификаторы (удаление пробелов и переносов строк) перед финальной сборкой.
  • Atlas Configuration: В папке assets/minecraft/atlases пропиши пути к своим текстурам, чтобы Minecraft загружал их одной большой картой (это экономит FPS игрокам).
  • Hash-кодирование: Если ты используешь кастомные звуки, убедись, что они в формате .ogg с битрейтом не выше 128kbps.

31. Кастомные интерфейсы: Fonts & Unicode

Чтобы в Aerhold были красивые меню или иконки в чате, мы используем систему перекрытия символов Юникода через ресурспак (внутри твоего JAR).

Файл шрифта (default.json):

{
  "providers": [
    {
      "type": "bitmap",
      "file": "aerhold:font/custom_gui.png",
      "ascent": 8,
      "chars": ["\uE001", "\uE002"]
    }
  ]
}

Использование:

Теперь символ \uE001 в титрах или названии предмета будет отображаться как твоя картинка (например, герб Одина или фон меню).

32. Кватернионы: Сложные вращения моделей

Если ты хочешь, чтобы твой 3D-моб плавно наклонялся при ходьбе или вращался по всем осям, обычных градусов недостаточно. Используются Кватернионы [x, y, z, w].

# Формула поворота на 90 градусов по оси Y:
# [0, sin(45°), 0, cos(45°)] -> [0, 0.707, 0, 0.707]

data merge entity @e[tag=model,limit=1] {
  transformation: {
    left_rotation: [0f, 0.707f, 0f, 0.707f]
  }
}

Совет: используй онлайн-калькуляторы "Euler to Quaternion", чтобы не считать это вручную.

33. Интеграция: Тэги и Совместимость

Чтобы твой датапак-мод "видел" предметы из других модов (например, если игрок поставил JEI или REI), всегда используй Тэги (#) вместо прямых ID.

  • Вместо minecraft:iron_ingot используй #c:iron_ingots (общепринятый стандарт Fabric/Forge для всех модов).
  • Это позволит твоим рецептам в Aerhold принимать железо из любого другого технического мода.

34. Защита от взлома (Server-side Checks)

Поскольку твой код работает на сервере, ты можешь проверять легитимность действий игрока. Например, если игрок наносит слишком много урона без нужных тегов:

# Проверка на киллауру / читы на урон
execute as @a[tag=!admin] at @s if score @s damage_dealt matches 100.. run kick @s "Suspicious Damage"

1. Выдача эффектов (Команда Effect)

Базовая выдача эффекта игроку. 1 0 true — это 1 секунда, 0 уровень (первый), скрытие частиц.

# Выдать силу на 1 секунду без пузырьков
effect give @s minecraft:strength 1 0 true

# Снять все эффекты с игрока
effect clear @s

2. Эффект при ношении предмета (Атрибуты)

В новых версиях (1.21+) можно давать ХП или скорость прямо через предмет, без команд в тике!

# Выдать шлем, который дает +2 сердца (4 хп)
/give @p netherite_helmet[attribute_modifiers={modifiers:[{type:"minecraft:generic.max_health",amount:4,slot:"head",id:"aerhold:health_boost",operation:"add_value"}]}]

3. Проверка предмета в руке (SelectedItem)

Если хочешь, чтобы эффект давался, только когда игрок ДЕРЖИТ меч:

# В tick.mcfunction: даем скорость, если в главной руке наш меч
execute as @a[nbt={SelectedItem:{id:"minecraft:netherite_sword",components:{"minecraft:custom_data":{aerhold_weapon:1}}}}] run effect give @s minecraft:speed 1 1 true

4. Выполнение команды от лица моба/игрока

Самая важная команда — execute. Она позволяет менять "точку сборки" кода.

КомандаЧто делает
execute as @e[type=zombie]Запуск кода ОТ ЛИЦА всех зомби
execute at @sЗапуск В ТОЧКЕ, где стоит игрок
execute if entity @p[distance=..5]Если игрок в радиусе 5 блоков
execute anchored eyesСчитать координаты от ГЛАЗ, а не от ног

5. Логика "Если... То..." (Execute Store)

Например, если хочешь ударить молнией того, кто ударил тебя (через скорборд):

# Проверка: если игрок получил урон (нужен скорборд damage_taken)
execute as @a[scores={damage_taken=1..}] at @s run summon lightning_bolt ~ ~ ~
scoreboard players set @a[scores={damage_taken=1..}] damage_taken 0

6. Звуки (Playsound)

Чтобы магия чувствовалась, добавляй звук. ambient — категория, @s — кому слышно, ~ ~ ~ — где, 1 — громкость, 1 — тон.

playsound minecraft:entity.warden.sonic_boom ambient @a ~ ~ ~ 1 0.5

1. Создание переменной (Scoreboard)

Сначала нужно создать саму «ячейку памяти». Это делается один раз (в файле load.mcfunction).

# Создать задачу для кулдауна (тип dummy — меняется только командами)
scoreboard objectives add cooldown_timer dummy

# Создать задачу для маны (автоматически растет, если хочешь)
scoreboard objectives add mana_points dummy

2. Логика Кулдауна (Перезарядки)

Представь: игрок нажал на предмет, сработала магия, и мы поставили ему счетчик 100 (5 секунд). Каждое мгновение мы отнимаем по 1. Пока счетчик больше 0 — магия не сработает.

Шаг А: Уменьшение счетчика (в tick.mcfunction):

# Уменьшаем таймер у всех, у кого он больше 0
scoreboard players remove @a[scores={cooldown_timer=1..}] cooldown_timer 1

Шаг Б: Проверка перед использованием:

# Сработать (молния), только если таймер = 0 и игрок держит меч
execute as @a[scores={cooldown_timer=0}, nbt={SelectedItem:{id:"minecraft:iron_sword"}}] at @s run summon lightning_bolt ~ ~ ~

# Сразу после срабатывания ставим кулдаун 100 тиков (5 сек)
execute as @a[scores={cooldown_timer=0}, nbt={SelectedItem:{id:"minecraft:iron_sword"}}] run scoreboard players set @s cooldown_timer 100

3. Математика в Скорбордах (Operations)

Ты можешь складывать, вычитать и умножать очки игроков. Например, передать очки одного игрока другому.

ОперацияПример кода
Приравнять (=)scoreboard players operation @s mana = @p old_mana
Сложить (+=)scoreboard players operation @s coins += @p bonus
Вычесть (-=)scoreboard players operation @s hp -= @e[limit=1] damage

4. Вывод очков на экран (Sidebar)

Чтобы игрок видел свою ману или КД справа на экране или под ником.

# Показать список справа (sidebar)
scoreboard objectives setdisplay sidebar mana_points

# Показать под ником (belowName)
scoreboard objectives setdisplay belowName mana_points

5. Кулдаун через предмет (Версия 1.21+)

В новых версиях можно сделать визуальную «шторку» на предмете, как у эндер-перла, без скорбордов!

# Наложить кулдаун 100 тиков на все железные мечи игрока
item cooldown @s minecraft:iron_sword 100

💡 Советы по оптимизации и визуалу

1. Забытые "Тики" (Время в игре)

Помни: 1 секунда = 20 тиков. Если хочешь кулдаун на 10 секунд — ставь в скорборд 200.

Если хочешь, чтобы частицы летели медленно — ставь скорость 0.01, если быстро — 0.5+.

2. Скрывай лишнее (Clean UI)

В крутых проектах вроде Aerhold не должно быть лишнего спама в чате и пузырьков.

# Всегда пиши true в конце эффекта, чтобы скрыть частицы зелий
effect give @s strength 1 0 true

# Отключи системные сообщения в чате (когда команда срабатывает)
gamerule commandBlockOutput false
gamerule sendCommandFeedback false

3. Индикатор перезарядки (Actionbar)

Чтобы игрок не гадал "почему магия не работает", выводи время КД прямо над его инвентарем:

title @a actionbar ["",{"text":"Перезарядка: ","color":"gray"},{"score":{"name":"@s","objective":"cooldown_timer"},"color":"red"}]
