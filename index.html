Раздел I Основы: Java, версии и pack.mcmeta

Связь Java и Minecraft

Майнкрафт написан на Java. Каждая крупная версия игры требует определённую версию JDK. Если запускать 1.21 с Java 8 — ничего не запустится.

Файл pack.mcmeta

Лежит в корне датапака. Без него папка — просто набор файлов, игра её не увидит.

{
  "pack": {
    "description": "§6Aerhold Project §7by KERDYS\n§bVersion 1.0",
    "pack_format": 48
  }
}
Подсказка
§6 — золото, §b — голубой, §c — красный. \n — перенос строки. Описание в меню выбора датапаков будет цветным!

Цветовые коды §

  • §0 — чёрный
  • §6 — золотой
  • §b — голубой
  • §c — красный
  • §e — жёлтый
  • §f — белый
  • §l — жирный
  • §o — курсив

Поддерживаемые форматы датапака

Начиная с 1.20.2 можно задать диапазон совместимых версий через supported_formats, чтобы датапак работал на нескольких версиях сразу:

{
  "pack": {
    "pack_format": 48,
    "supported_formats": [48, 61]
  }
}
Раздел II Анатомия и структура датапака

Файловое дерево

Датапак — это ZIP-архив или обычная папка. Игра читает их одинаково. Вот полная структура профессионального датапака:

my_datapack/
├── pack.mcmeta          ← обязательный манифест
├── pack.png             ← иконка (64×64 PNG)
└── data/
    ├── minecraft/       ← переопределение ванильного контента
    │   ├── tags/
    │   │   ├── functions/
    │   │   │   ├── tick.json     ← каждый тик
    │   │   │   └── load.json     ← при загрузке
    │   │   └── worldgen/biome/
    │   └── recipes/
    └── aerhold/         ← твой namespace
        ├── functions/   ← .mcfunction файлы
        ├── worldgen/
        │   ├── biome/
        │   ├── dimension/
        │   ├── dimension_type/
        │   ├── noise_settings/
        │   ├── density_function/
        │   └── placed_feature/
        ├── recipes/
        ├── advancements/
        ├── loot_tables/
        ├── predicates/
        ├── damage_type/
        ├── enchantment/ ← 1.21+
        └── structures/

Namespace: правило двух папок

В папке data всегда должно быть минимум ДВЕ папки:

minecraft/

Здесь ты переопределяешь ванильные вещи: рецепты, теги биомов, функции tick/load. Файлы с тем же именем заменят оригиналы.

aerhold/ (твой namespace)

Здесь ты создаёшь своё: функции, биомы, структуры, рецепты. Вызов: /function aerhold:cool_effect

Важно
Namespace пишется строчными латинскими буквами, только a–z, 0–9, _ и -. Пробелы и CamelCase — нельзя.
Раздел III Генерация мира

Noise Settings: основа рельефа

Самый главный файл генерации. Он задаёт физические параметры мира и входную точку для всех функций плотности.

{
  "sea_level": 63,
  "disable_mob_generation": false,
  "aquifers_enabled": true,
  "ore_veins_enabled": true,
  "noise": {
    "min_y": -64,
    "height": 384,
    "size_horizontal": 1,
    "size_vertical": 2
  },
  "noise_router": { /* ... см. раздел V */ },
  "surface_rule": { /* ... блоки на поверхности */ },
  "spawn_target": []
}

Noise Router и Density Functions Advanced

В версиях 1.19+ рельеф строится на каскаде функций плотности. Noise Router — это «карта маршрутов», которая связывает математические функции с конкретными аспектами генерации:

Ключевые узлы роутера

  • barrier — граница твёрдого блока
  • fluid_level_flooded — уровень воды в пещерах
  • vein_toggle — переключатель рудных жил
  • continents — глобальный шум суши/океана
  • erosion — эрозия: плоские/горные зоны
  • ridges — хребты и пики
  • initial_density_without_jaggedness — базовая форма рельефа
  • final_density — финальная плотность блоков

Типы Density Functions

  • minecraft:noise — базовый шум (Perlin/Simplex)
  • minecraft:shifted_noise — шум со смещением (реки)
  • minecraft:weird_scaled_sampler — хребты гор
  • minecraft:interpolated — трилинейная интерполяция
  • minecraft:blend_density — смешивание при загрузке старых чанков
  • minecraft:add, minecraft:mul — математика
  • minecraft:min, minecraft:max — фильтрация
  • minecraft:spline — кривые Безье для рельефа

Пример составной функции плотности

{
  "type": "minecraft:interpolated",
  "argument": {
    "type": "minecraft:shifted_noise",
    "noise": "minecraft:clay_bands_offset",
    "xz_scale": 1.5,
    "y_scale": 0.0,
    "shift_x": "minecraft:island_noise",
    "shift_y": 0,
    "shift_z": "minecraft:island_noise"
  }
}

Splines: кривые фильтрации рельефа

Splines — это графики перевода «значения шума» в «высоту горы». Именно они делают Minecraft горы горами, а не просто случайными холмами.

Принцип работы
Если terrain_estimator = 0.5 — сделать гору острой. Если -0.2 — превратить в каньон. Кривая задаёт эту зависимость. 
"points": [
  { "location": -1.0, "value": 0.1, "derivative": 0.0 },
  { "location": -0.5, "value": 0.2, "derivative": 0.5 },
  { "location":  0.0, "value": 0.5, "derivative": 1.2 },
  { "location":  0.5, "value": 0.9, "derivative": 0.8 },
  { "location":  1.0, "value": 1.2, "derivative": 0.0 }
]

derivative — это «скорость изменения» в данной точке. Высокое значение = крутой склон.

Создание измерения: Нифльхейм Advanced

Два обязательных файла: dimension_type (физика) и dimension (генератор).

dimension_type/niflheim.json

{
  "ultrawarm": false,
  "natural": true,
  "coordinate_scale": 1.0,
  "has_skylight": true,
  "has_ceiling": false,
  "ambient_light": 0.1,
  "monster_spawn_light_level": 0,
  "monster_spawn_block_light_limit": 0,
  "infiniburn": "#minecraft:infiniburn_overworld",
  "effects": "minecraft:overworld",
  "min_y": -64,
  "height": 384,
  "logical_height": 384
}

Параметры на заметку

  • ultrawarm — вода испаряется (как в Нижнем)
  • ambient_light — фоновое освещение 0–1
  • effects — небо, туман, музыка. Используй minecraft:the_end для лилового неба
  • has_skylight — есть ли дневной цикл
  • coordinate_scale — 8 для «нижнего» масштаба
Совет
Для Нифльхейма: ambient_light: 0.05, effects: "minecraft:the_end" — получишь вечную синюю ночь.

Биомы: размещение структур Worldgen

Чтобы скандинавские постройки (драккары, алтари) не спавнились в пустыне, используются Structure Sets и Biome Tags.

Биомный фильтр (тег)

/* data/minecraft/tags/worldgen/
   biome/has_structure/altar.json */
{
  "replace": false,
  "values": [
    "#minecraft:is_taiga",
    "#minecraft:is_mountain",
    "minecraft:snowy_plains"
  ]
}

Расположение структур

/* aerhold/worldgen/
   structure_set/viking_villages.json */
{
  "structures": [{
    "structure": "aerhold:viking_village",
    "weight": 1
  }],
  "placement": {
    "type": "minecraft:random_spread",
    "spacing": 32,
    "separation": 8,
    "salt": 12345678
  }
}

spacing — среднее расстояние между структурами в чанках. separation — минимальное. salt — уникальное число, чтобы разные структуры не накладывались.

Раздел IV Функции, логика и команды

Execute: главная команда датапака

execute — это «переключатель контекста». Она меняет, от кого, где и при каком условии выполняется код.

СинтаксисЧто делает

execute at @sЗапуск В ТОЧКЕ исполнителя

Выдача эффектов и предметов

Команда effect

# Сила на 1 сек без частиц
effect give @s minecraft:strength 1 0 true

# Снять все эффекты
effect clear @s

# Эффект только если держит меч
execute as @a[nbt={SelectedItem:{
  id:"minecraft:netherite_sword"}}]
run effect give @s minecraft:speed 1 1 true

Атрибуты на предмете (1.21+)

# Шлем +2 сердца
/give @p netherite_helmet[
  attribute_modifiers={modifiers:[{
    type:"minecraft:generic.max_health",
    amount:4,
    slot:"head",
    id:"aerhold:health_boost",
    operation:"add_value"
  }]}
]
Чистый интерфейс
Всегда пиши true в конце effect give — это скроет частицы зелий. Также отключай спам в консоли:
gamerule commandBlockOutput false
gamerule sendCommandFeedback false

Tick & Load: сердце датапака

data/minecraft/tags/functions/tick.json

{
  "values": [
    "aerhold:tick/main"
  ]
}

Функция main запускается КАЖДЫЙ тик (20 раз/сек). Держи её лёгкой — только диспетчер, который вызывает другие функции.

data/minecraft/tags/functions/load.json

{
  "values": [
    "aerhold:init/load"
  ]
}

Запускается при /reload и загрузке мира. Здесь инициализируй Scoreboard и Storage, ставь начальные значения.

Оптимизация
Никогда не делай сложных вычислений прямо в tick-функции. Используй вложенные функции и вызывай их только по условию. 1 тик = 1/20 секунды = 50 мс. Опоздаешь — TPS сервера упадёт.
Раздел V Scoreboard: переменные и кулдауны

Создание переменных в load.mcfunction

# Таймер перезарядки
scoreboard objectives add cooldown_timer dummy

# Мана игрока
scoreboard objectives add mana_points dummy

# Отслеживать урон автоматически
scoreboard objectives add damage_taken minecraft.custom:minecraft.damage_taken

Логика кулдауна: система перезарядки

В tick.mcfunction — уменьшение таймера

# Уменьшаем у всех, у кого > 0
scoreboard players remove @a[
  scores={cooldown_timer=1..}
] cooldown_timer 1

В функции способности

# Сработать только если КД = 0
execute as @a[
  scores={cooldown_timer=0},
  nbt={SelectedItem:{id:"minecraft:iron_sword"}}
] at @s run summon lightning_bolt ~ ~ ~

# Поставить КД 5 секунд
execute as @a[
  scores={cooldown_timer=0},
  nbt={SelectedItem:{id:"minecraft:iron_sword"}}
] run scoreboard players set @s cooldown_timer 100
Запомни
1 секунда = 20 тиков. Кулдаун 5 сек = 100, 10 сек = 200, 1 мин = 1200.

Раздел VI Command Storage: база данных

Storage vs Scoreboard

Scoreboard

  • Только целые числа
  • Быстрые вычисления
  • Идеален для таймеров, счётчиков, HP
  • Виден в /scoreboard players list

Storage

  • JSON — строки, массивы, объекты
  • Не исчезает после выхода из игры
  • Идеален для квестов, инвентаря персонажа, RPG-данных
  • Доступен из любой функции
# Записать данные квеста в хранилище
data modify storage aerhold:players CurrentQuest set value {
  id:"kill_frost_giant",
  progress:0,
  reward:"iron_ingot"
}

# Проверить активен ли квест
execute if data storage aerhold:players {
  CurrentQuest:{id:"kill_frost_giant"}
} run say Квест: Убить Ледяного Великана — активен!

# Передать данные из NBT сущности в storage
data modify storage aerhold:world SpawnPos set from entity @e[tag=spawn_marker,limit=1] Pos
Раздел VII Predicates: умные условия

Предикаты против Execute

Предикаты — это JSON-файлы с условиями. Они работают быстрее чем длинные цепочки execute if, их легко переиспользовать и комбинировать.

Предикат: держит Мьёльнир

/* aerhold/predicates/
   is_holding_mjolnir.json */
{
  "condition": "minecraft:entity_properties",
  "entity": "this",
  "predicate": {
    "equipment": {
      "mainhand": {
        "items": ["minecraft:netherite_axe"],
        "predicates": {
          "minecraft:custom_data": {
            "aerhold_type": "mjolnir"
          }
        }
      }
    }
  }
}

Предикат: открыт шторму

/* aerhold/predicates/
   is_exposed_to_storm.json */
{
  "condition": "minecraft:all_of",
  "terms": [
    {
      "condition": "minecraft:weather_check",
      "is_thundering": true
    },
    {
      "condition": "minecraft:location_check",
      "predicate": {
        "can_see_sky": true,
        "biome": "#aerhold:is_frozen"
      }
    }
  ]
}

Использование в функции

execute as @a at @s if predicate aerhold:is_holding_mjolnir run function aerhold:abilities/thunder_strike
execute as @a at @s if predicate aerhold:is_exposed_to_storm run function aerhold:weather/freeze_player
Раздел VIII Raycasting: магические лучи и копья

Алгоритм рейкастинга

Рейкастинг — это рекурсивная функция, которая «летит» вперёд по 0.5 блока за вызов, пока не встретит блок или сущность. Так работают скандинавские копья, молнии и огненные шары.

Шаги алгоритма

  1. Игрок вызывает aerhold:ray/start
  2. Спавним маркер у глаз игрока
  3. Маркер телепортируется на 0.5 блока вперёд
  4. Проверяем: блок = воздух? → продолжаем
  5. Есть сущность рядом? → наносим урон
  6. Если блок твёрдый → останавливаемся

ray/start.mcfunction

summon marker ~ ~1.6 ~ {
  Tags:["aerhold_ray","new_ray"]
}
execute as @e[tag=new_ray] at @s run function aerhold:ray/travel
tag @e[tag=new_ray] remove new_ray

ray/travel.mcfunction

# Двигаемся вперёд на 0.5 блока
tp @s ^ ^ ^0.5

# Если рядом есть цель — наносим урон и удаляем луч
execute as @s at @s if entity @e[type=!player,type=!marker,distance=..1.2] run function aerhold:ray/hit

# Если впереди воздух — продолжаем рекурсию (макс 40 блоков)
execute as @s at @s if block ~ ~ ~ air run function aerhold:ray/travel

# Если блок — стоп
execute as @s at @s unless block ~ ~ ~ air run kill @s

ray/hit.mcfunction

# Наносим урон ближайшей сущности
damage @e[type=!player,type=!marker,distance=..1.2,limit=1] 8 aerhold:frost

# Эффекты попадания
particle minecraft:snowflake ~ ~ ~ 0.3 0.3 0.3 0.05 30
playsound minecraft:block.glass.break block @a ~ ~ ~ 1 1.5

kill @s
Раздел IX Item Display и Interaction: кастомные мобы

Современный стандарт: Display + Interaction

Забудь про Armor Stand. Теперь используют связку: Item Display для визуала и Interaction для хитбокса.

Визуал — Item Display

summon item_display ~ ~ ~ {
  item:{
    id:"minecraft:netherite_axe",
    Count:1b,
    components:{
      "minecraft:custom_model_data":1001
    }
  },
  transformation:{
    left_rotation:[0f,0f,0f,1f],
    right_rotation:[0f,0f,0f,1f],
    translation:[0f,0.5f,0f],
    scale:[2f,2f,2f]
  },
  interpolation_duration:10,
  Tags:["aerhold_mob","visual"]
}

Хитбокс — Interaction

summon interaction ~ ~ ~ {
  width:1.5f,
  height:3.0f,
  attack:{
    player:"",
    timestamp:0L
  },
  Tags:["aerhold_mob","hitbox"]
}

# Проверка удара по сущности
execute as @e[tag=hitbox] if entity @s[
  nbt={attack:{player:""}}
] run function aerhold:mob/take_damage
Совет
Связывай Display и Interaction тегом UUID или ridable-системой. Оба должны телепортироваться вместе в tick-функции: execute as @e[tag=visual] at @s run tp @e[tag=hitbox,distance=..10,limit=1] ~ ~ ~
Раздел X Анимация и кватернионы

Transformation Interpolation

Вместо сотен команд tp используется интерполяция. Игра сама рассчитывает плавный переход.

Автогенерация файлов: Python-скрипт

Когда нужно создать 500 рецептов — пишешь скрипт, а не файлы вручную.

# generate_recipes.py
import json
import os

metals = ["iron", "gold", "netherite", "diamond", "emerald"]
weapons = ["sword", "axe", "pickaxe", "shovel"]

for metal in metals:
    for weapon in weapons:
        recipe = {
            "type": "minecraft:crafting_shaped",
            "pattern": ["XX", "X ", " X"],
            "key": {
                "X": f"minecraft:{metal}_ingot"
            },
            "result": {
                "id": f"aerhold:{metal}_{weapon}",
                "count": 1
            }
        }
        path = f"data/aerhold/recipes/{metal}_{weapon}.json"
        os.makedirs(os.path.dirname(path), exist_ok=True)
        with open(path, "w") as f:
            json.dump(recipe, f, indent=2, ensure_ascii=False)

Раздел XXII Макросы: динамические функции

Что такое Macro? 1.20.2+

Макросы — это шаблонные переменные $(name) внутри .mcfunction. Ты передаёшь значения при вызове, и команды подставляют их прямо в текст. Это полностью меняет то, как пишутся датапаки: вместо 50 одинаковых функций — одна с параметрами.

Правило
Строка с $ в начале — это макро-строка. Если в функции есть хоть одна такая строка, она называется «macro function» и вызывается иначе: function my:fn {key:"value"} или function my:fn with storage …

Передача аргументов: три способа

1. Прямо в команде (NBT compound)

# Вызов с литеральными значениями
function aerhold:give_item {
  item: "minecraft:diamond_sword",
  name: "Меч Одина",
  color: "gold"
}

# aerhold/functions/give_item.mcfunction
$give @s $(item){display:{Name:'{"text":"$(name)","color":"$(color)"}'}}
$tellraw @s {"text":"Получено: $(name)","color":"$(color)"}

2. Из Storage

# Сначала заполняем хранилище
data modify storage aerhold:macro TpData set value {
  x: 100,
  y: 64,
  z: -200,
  msg: "Телепорт на базу"
}

# Вызываем функцию с данными из storage
function aerhold:teleport with storage aerhold:macro TpData

# aerhold/functions/teleport.mcfunction
$tp @s $(x) $(y) $(z)
$title @s actionbar {"text":"$(msg)","color":"aqua"}

3. Из NBT сущности или блока

# Из NBT маркера
function aerhold:spawn_mob with entity @e[tag=spawn_config,limit=1] data

# Из NBT блока (chest, command block…)
function aerhold:configure with block 0 64 0 Items[0].tag

# aerhold/functions/spawn_mob.mcfunction
$summon $(mob_type) $(x) $(y) $(z) {Tags:["$(faction)"],Health:$(health)f}

Классика — сохранить точку и вернуться к ней. Без макросов это требовало костылей через Scoreboard и три отдельные команды tp.

execute store result storage aerhold:waypoints HomeFlat.z double 1 run data get entity @s Pos[2]

# aerhold/functions/tp_home.mcfunction

Раздел XXIV Advancements: детекторы событий

Зачем нужны Advancements в датапаках?

Большинство разработчиков думают, что Advancements — только для достижений. Это не так. Главная их роль в датапаках — быть детекторами событий, которые срабатывают на конкретные действия игрока и запускают функции.

Что умеет детектировать

  • Убийство конкретного моба
  • Подбор предмета с кастомными данными
  • Крафт предмета
  • Вход в биом / измерение
  • Получение урона от типа
  • Использование предмета (еда, зелья)
  • Выстрел из лука / арбалета
  • Первый тик после захода в мир
  • Любой кастомный NBT на игроке

Скрытое достижение-детектор

{
  "criteria": {
    "trigger": {
      "trigger": "minecraft:killed_by_crossbow",
      "conditions": {
        "unique_entity_types": 1
      }
    }
  },
  "rewards": {
    "function": "aerhold:events/crossbow_kill"
  },
  "display": {
    "hidden": true,
    "title": "",
    "description": ""
  }
}
Важно: Revoke
После срабатывания достижение остаётся «выполненным». Если нужен повторный триггер — в rewards.function первым делом отзывай его: advancement revoke @s only aerhold:events/my_trigger

NBT — это формат бинарных данных Minecraft. Каждое значение имеет суффикс-тип. Без правильного суффикса данные не запишутся или запишутся не так.

Путь к данным — это строка, которая указывает куда именно читать/писать в NBT-дереве. Используется в data get, data modify, execute store.

Раздел XXXVIII Schedule: отложенные и цикличные функции

Команда schedule

schedule позволяет запустить функцию через N тиков. Это мощный инструмент для таймаутов, анимаций, задержанных эффектов и замены тяжёлых тик-функций.

СинтаксисЧто делает
schedule function f:n 20tЗапустить через 20 тиков (replace режим)
schedule function f:n 20t appendДобавить в очередь (не заменять)
schedule function f:n 1sЧерез 1 секунду
schedule function f:n 2dЧерез 2 игровых дня
schedule clear f:nОтменить запланированную функцию

Jigsaw: процедурные структуры

Jigsaw — система, которая собирает структуры из кусочков по правилам соединений. Идеально для деревень викингов, подземелий, дорог.

Принцип работы

  1. Каждый «кусочек» (pool element) — это файл структуры (.nbt)
  2. В структуре стоят блоки jigsaw с настройками соединений
  3. Блок указывает: «сюда подходит кусочек из пула X с соединением Y»
  4. Игра рекурсивно достраивает структуру до лимита глубины

        "type": "minecraft:single_pool_element",
        "location": "aerhold:village/house_small",
        "projection": "rigid"
      }
    },
    {
      "weight": 1,
      "element": {
        "type": "minecraft:single_pool_element",
        "location": "aerhold:village/longhouse",
        "projection": "terrain_matching"
      }
    }
  ]
}
Совет
projection: "terrain_matching" адаптирует кусочек под рельеф земли. rigid — жёсткое размещение без адаптации. Для дорог и мостов используй terrain_matching.
